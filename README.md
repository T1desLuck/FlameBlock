# FlameBlock — блокировщик рекламы (v1.2.1)

Расширение для Chrome и Opera (Manifest V3, один и тот же код для обоих браузеров).
Полностью автономное: все настройки хранятся локально в браузере, ничего никуда не отправляется.
Локализовано на 12 языков. В v1.2.1 усилено покрытие рекламы Яндекса/Mail.ru/VK/Unity Ads
(подробности — SPEC.md, раздел 3).

Подробности — в двух документах рядом:
- **SPEC.md** — полная техническая спецификация: архитектура, что каждый файл делает,
  откуда взяты списки, как считается бейдж, как устроена локализация и т.д.
- **RESEARCH.md** — результаты реального веб-исследования по состоянию блокировщиков на
  середину 2026 года и честная оценка, какие продвинутые техники применимы на Chrome/Opera,
  какие невозможны в принципе (платформенные ограничения), и что из этого добавлено сюда.

## Возможности

- Сетевая блокировка рекламы и трекеров через `declarativeNetRequest` — быстрый нативный
  движок браузера, минимальная нагрузка на систему.
- Косметическая фильтрация — CSS-паттерны известных рекламных движков + текстовая
  эвристика (короткие метки «Реклама»/«Sponsored» → скрывается контейнер-обёртка).
- Эвристика по размеру iframe (300×250, 728×90 и т.д. с чужого домена) — выключена по
  умолчанию, включается в настройках.
- Стелс-режим — сетевая блокировка известных анти-adblock скриптов
  (`rules/antiadblock.json`) + лёгкий скриптлет прямо в контексте страницы
  (`scriptlets.js`, MAIN world) с гейтом на `window.open` против попандеров.
- Счётчик заблокированного на значке расширения (кружок `#b52b0e`).
- Разрешение рекламы для конкретного сайта (тумблер в попапе, запоминается).
- Свои правила (блок / разрешение по домену) в расширенных настройках.
- **Локализация на 12 языков** — русский (запасной по умолчанию), английский, китайский
  (упрощённый), испанский, хинди, арабский (с поддержкой RTL), португальский (Бразилия),
  японский, корейский, немецкий, французский, индонезийский. Автоопределение по языку
  браузера, ручной выбор в настройках запоминается. Подробно — в SPEC.md, раздел 6.
- Правила собраны из официального community-списка EasyList/EasyPrivacy
  (github.com/easylist/easylist) плюс вручную проверенный список крупных сетей.

## Установка для тестирования (разработчик)

Одинаково для Chrome и Opera — оба на Chromium и понимают один и тот же пакет.

**Chrome:** `chrome://extensions` → включить «Режим разработчика» (правый верхний угол) →
«Загрузить распакованное расширение» → выбрать папку `flameblock`.

**Opera:** `opera://extensions` → включить «Режим разработчика» → «Загрузить распакованное
расширение» → выбрать папку `flameblock`.

```
flameblock/

├── RESEARCH.md              # разбор техник блокировки и их применимости
├── manifest.json          # Manifest V3 — общий для Chrome и Opera
├── background.js          # service worker: бейдж, вайт-лист, свои правила, скриптлет
├── content.js              # косметическая фильтрация + эвристики на странице
├── scriptlets.js            # MAIN-world скриптлет (гейт на window.open)
├── i18n.js                  # определение языка + применение переводов к DOM
├── common.js               # общие настройки/хелперы (background + popup + options)
├── _locales/                # переводы на 12 языков (ru, en, zh_CN, es, hi, ar, pt_BR, ja, ko, de, fr, id)
├── rules/
│   ├── ads.json             # блокировка рекламных сетей
│   ├── trackers.json        # блокировка трекеров/аналитики
│   └── antiadblock.json     # стелс-режим против анти-adblock скриптов
├── popup/                   # всплывающее окно (клик по иконке)
├── options/                 # расширенные настройки
├── icons/
└── generate_rules.py        # скрипт, которым собраны rules/*.json — можно перезапустить
                              # после обновления easylist-master/, чтобы обновить списки
```

## Публикация в магазины — в чём разница

Код один и тот же для обоих магазинов, разница только в самой процедуре публикации:

- **Chrome Web Store** (chrome.google.com/webstore/devconsole) — нужен аккаунт разработчика
  с разовым взносом $5 (платится один раз навсегда, покрывает до 20 расширений на аккаунт),
  дальше загружается .zip этой папки. Модерация обычно занимает от нескольких часов до
  нескольких дней.
- **Opera Add-ons** (addons.opera.com) — отдельный аккаунт разработчика, регистрация
  бесплатная, без взноса. Загружается тот же .zip, manifest.json подходит без изменений —
  модераторы Opera проверяют расширение и либо публикуют, либо возвращают с замечаниями.
  Opera также умеет напрямую ставить расширения из Chrome Web Store, так что публикация
  там может быть достаточной сама по себе.

Для быстрой ручной раздачи (не через магазин) можно просто присылать людям .zip — им нужно
будет включить «Режим разработчика» и загрузить распакованное расширение, как описано выше.

## Известные ограничения этой версии

Подробный, честный разбор — в RESEARCH.md, коротко:

- Список правил держим ~24 000 — с запасом ниже гарантированного Chrome лимита в 30 000
  (`GUARANTEED_MINIMUM_STATIC_RULES`), чтобы расширение работало стабильно вне зависимости
  от того, что ещё стоит у пользователя. Можно расширять дальше — это следующий шаг.
- Стелс-режим — это список известных сервисов-детекторов + скриптлет-гейт на popup, а не
  универсальный обход любой защиты. Сайты меняют детекторы, список придётся пополнять.
- CNAME uncloaking (раскрытие трекеров за первым лицом) невозможен на Chrome/Opera в
  принципе — у Chromium нет DNS API для расширений, это есть только в Firefox. Не наша
  недоработка, платформенное ограничение (см. RESEARCH.md, раздел 2).
- Эвристика по размеру iframe выключена по умолчанию — у неё выше риск ложных
  срабатываний, чем у списков.
